01 — Color palette
Decision — Updated Jun 2026
Butter Yellow + Richer Periwinkle on Near-White Cream

Original palette: #E2ED84 (chartreuse) + #ABBAF9 (pale periwinkle). Updated: #FAE96A (butter yellow) — warmer, less acidic, feels more personal. #7891F7 (richer periwinkle) — 10% deeper, text is now legible against cream without losing the blue-violet hue. Cream shifted to #FEFEF9 — barely perceptible but removes the slight yellow cast that clashed with the new yellow accent.

Rule: Colors touch less than 10% of the page surface. Periwinkle = chips + tape. Yellow = highlights, stickers, now-dot only.

02 — Typography
Decision
Bricolage Grotesque + Caveat

Tested: Fraunces (too delicate against bold colors), Cormorant Garamond (too close to galpal), Instrument Serif (too editorial/polished), Darker Grotesque (strong alternative). Chosen: Bricolage Grotesque 800 — chunky, warm, has personality bumps that stop it feeling corporate. Caveat for all annotations, captions, stat asides.

Rule: Headings always dark #111800. Periwinkle never on heading text. Bricolage 800 for display, 400 for body, 500 for inline story text.

03 — Text highlight styles
Chosen — Marker wash
people love.

Gradient fades in/out at left and right edges — like a real marker stroke. Used when NO eyebrow chip is present in the section. Butter yellow at 45% opacity.

CSS: linear-gradient(to right, transparent 0%, rgba(250,233,106,0.5) 7%, … transparent 100%)

Chosen — Underline
people.

Used when an eyebrow chip IS present in the same section — avoids double emphasis. 3px border-bottom, butter yellow at 85% opacity.

Rule: Never use marker wash AND underline in the same section. Never use either with a chip unless it's underline.

04 — Background texture
Decision
Dot grid, 4.4% opacity

Tested: graph paper (too structured), ruled lines + margin (too diary), crosshatch (too faint). Chosen: dot grid — most breathing room, most like a Leuchtturm notebook. Opacity pulled back so dots are ambient, not noisy.

CSS: radial-gradient(circle, rgba(0,0,0,0.044) 1.2px, transparent 1.2px) · background-size: 22px 22px

Constraint
Cream all the way through

No dark sections anywhere. Every section background is the same near-white cream #FEFEF9. Section breaks use a single 1px line at rgba(0,0,0,0.07) — a whisper, not a border.

Rule: If a section feels heavy, remove color and add space. Never add a dark background.

05 — Eyebrow chip
Component
Rotated periwinkle label tag
Behavioral PM · Founder

Background #7891F7, white text, 8px Bricolage 700, all-caps, letter-spacing 0.28em, border-radius 5px, rotate(-2deg), box-shadow 2px 2px 0 rgba(0,0,0,0.08).

Rule: ONE per section maximum. When chip is present → use underline for text emphasis. No wash highlight allowed in same section.

06 — Tilt system

The tilt is the personality. Everything non-body has a slight rotation. Headings and body copy stay perfectly straight — that contrast is what makes the tilts feel intentional.

08 — Icon style
Chosen — Option B

Slightly irregular paths that feel freehand. Stroke weight 1.5px. Key/accent icon gets chartreuse fill or periwinkle stroke. Supporting icons are plain dark outline.

09 — Layout decisions per section
01
Hero — Two column with Polaroid cluster right
Chosen from Layout 2 (centered) reference. Two Polaroids only (not three — avoids overwhelming). No stat note in hero — 32% is reserved for the Philosophy section. CTAs are outline buttons.
rejected: folder/pocket (Layout 5) — too daring for first impression
03
Philosophy — Two column with pull quote and stat cards
The WhatsApp message story lives in a periwinkle pull-quote block. Stats float right in slightly tilted cards. This is where 32% lives — not in the hero.
04
Story — Lighter inline text (Bricolage 500, not 800)
Original Layout 4 used Bricolage 800 at 42px for the inline text — too heavy, too much emphasis. Reduced to 500 weight, 26px. Sticker chips for key phrases remain at 800.
fix: "bold font taking up too much space and emphasis overall"
05
Skills — Q&A format with "Yes." in periwinkle
The "Can she...? Yes." structure anticipates every hiring manager objection and answers it directly. "Yes." in periwinkle at 22px is the visual anchor per row.
06
Case Studies — Cards (Layout 3 reference)
Header with palette color background, badge, project name. Honest take in italic. Tags as pill chips. Same structure across all three projects.
08
Personal — Polaroid string with washi tape hangers
Photos hung from a thin horizontal line using washi tape strips (not clips/forks). Blue and green tape alternate. Each photo at a different rotation angle.
fix: "current fork type shape looks weird" → replaced with washi tape strips
SP
Social Proof — Chat bubble format
iMessage/WhatsApp-style bubbles alternating left (periwinkle tint) and right (butter yellow tint). Feels like a real conversation rather than formal testimonials.
11 — Animations & interactions
A
Beyond Work — Continuous marquee carousel (rAF loop)
All 13 photo units are duplicated in HTML (aria-hidden on clones) to create a seamless loop. A requestAnimationFrame loop translates the row left at 1.2px/frame, resetting to 0 when the first half scrolls out. Hover pauses the loop. CSS animation was not used — overflow:hidden on the parent created a will-change compositing conflict that silently killed the animation.
CSS animation failed silently; rAF is the reliable path here
B
Philosophy stat cards — Per-card hover tilt via initGroupTilt
The existing initGroupTilt function was extended to accept a hoverDeltas array. On mouseenter each card applies its delta on top of its base rotation; on mouseleave the delta resets to 0. Card 1: −3°, Card 2: +3°, Card 3: −3°. The group tilt on mousemove continues to work simultaneously. Transition: 0.15s ease-out already on .stat-card.
direction per card = intentional personality, not random jitter
C
Contact polaroids — Real photos + confetti on hover
Polaroid photo wells replaced with real <img> tags using object-fit:cover. overflow:hidden removed from #connect so hover scaling isn't clipped. Base size: cp1 77×91px, cp2 71×83px (original +5px). Hover: img width/height grows by 10px via CSS transition (not transform scale — pixel-based gives precise control). Confetti burst via canvas-confetti CDN fires on mouseenter at the image's viewport position. Palette: periwinkle + butter yellow + annotation blue.
overflow:hidden on container was silently clipping CSS scale transforms; canvas-confetti uses position:fixed canvas so it was immune
D
Contact polaroid sizing constraint
Max hover size (87×101px for cp1, 81×93px for cp2) is far below the hero polaroid size (224×264px and 204×240px). This is the absolute ceiling — contact polaroids should never compete with or exceed hero polaroid scale.
hierarchy: hero polaroids are the landmark; contact polaroids are a delight detail
10 — Non-negotiable rules (updated)
Color
  1. No dark background sections — cream (#FEFEF9) throughout
  2. Butter yellow only on one phrase per section, at 45% opacity
  3. Periwinkle (#7891F7) never on heading text — only chips, tape, dots
  4. One eyebrow chip per section maximum
  5. Chip present → underline only, no wash highlight
  6. Yellow label text (#6B5800) for env-label-y, env-skill-y, tl-year-now
Typography
  1. Headings always dark #111800 (never periwinkle)
  2. Body always Bricolage 400, never heavier
  3. Caveat for all handwritten/personal elements only
  4. Story inline text = Bricolage 500 at 26px (not 800)
Layout + Interaction
  1. Headings and body copy always perfectly straight (0°)
  2. Everything else has a slight tilt
  3. If a section feels heavy: remove color, add space
  4. No overflow:hidden on sections with transform hover effects
  5. CSS animations that fail silently → switch to rAF
  6. Contact polaroids never exceed hero polaroid size (224×264px)
